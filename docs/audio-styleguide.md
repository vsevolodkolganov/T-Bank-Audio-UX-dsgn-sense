# Audio UX Style Guide
T-Bank Audio Interface

This document defines the sonic design system.

---

# Earcon Principles

Earcons must be:

short  
distinct  
memorable  
emotionally neutral

Maximum duration:

400 ms

---

# Frequency Guidelines

| Signal | Frequency |
|------|------|
Listening | 600–1200 Hz |
Processing | 120–240 Hz |
Success | harmonic interval |
Error | descending tone |

Low frequencies communicate **system activity**.

Higher frequencies communicate **attention signals**.

---

# Volume Hierarchy

Voice > Earcon > Ambient loop

Recommended levels:

Voice = 0 dB  
Earcon = −6 dB  
Processing loop = −18 dB

---

# Sonic Consistency

All interface sounds must follow the same acoustic profile:

soft attack  
short decay  
minimal reverb

The interface should feel **calm and precise**.

---

# Voice Guidelines

Voice characteristics:

warm  
neutral accent  
medium tempo  
clear articulation

Avoid robotic tone.

Speech should feel like **a calm conversation**.

---

# Interaction Timing

Target response time:

150 ms

Delays longer than 300 ms should activate **processing pulse**.

---

# Accessibility

Audio UX must support:

visually impaired users  
elderly users  
noisy environments

All critical events must include **haptic confirmation**.
